[[req_collections_rc-time-definition]]
[width="90%",cols="2,6a"]
|===
^|*Requirement {counter:req-id}* |*/req/collections/rc-time-definition* 
^|A |Temporal geometries are either a date-time value or a time interval. The parameter value SHALL conform to the following syntax (using link:https://tools.ietf.org/html/rfc2234[ABNF]):

[source,java]
----
interval-closed     = date-time "/" date-time
interval-open-start = "../" date-time
interval-open-end   = date-time "/.."
interval            = interval-closed / interval-open-start / interval-open-end
datetime            = date-time / interval
----
^|B |The syntax of `date-time` is specified by link:https://tools.ietf.org/html/rfc3339#section-5.6[RFC 3339, 5.6].
^|C |Open ranges in time intervals at the start or end SHALL be supported using a double-dot (`..`).
^|D |The `datetime` parameter SHALL have the following characteristics (using an OpenAPI Specification 3.0 fragment):

[source,YAML]
----
name: datetime
in: query
required: false
schema:
  type: string
style: form
explode: false
----
